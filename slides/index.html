Polymarket is the biggest prediction market and it's fundamentally centralized. They decide what questions get listed, they run resolution, they hold your funds. If they get a subpoena or decide your market is inconvenient, it disappears. Callit makes every one of those operator roles into a smart contract function. There is no compliance team.

Architecture

Three ways to drive Callit. The web app connects to PWallet — the Polkadot app — through the Triangle Host API from Nova Sama. PWallet holds the accounts and signs. The CLI uses alloy and a plain Ethereum RPC connection with its own signers. And the MCP server exposes every contract function as a typed tool over stdio, so an LLM like Claude can create markets, buy shares, resolve, and claim — all through natural language, with no manual transaction signing. All three converge on the same deployed addresses. The same Solidity source is compiled twice — once with solc for standard EVM, once with resolc from Parity for PolkaVM, which runs RISC-V. Both are deployed. On top sits an eth-rpc proxy that translates standard Ethereum JSON-RPC into Substrate extrinsics — so viem, alloy, MetaMask all just work.

Market Lifecycle

A market goes through Open → Resolving → Proposed. From Proposed, two paths: if no one disputes within 24 hours it finalizes and the resolver gets their bond back. If disputed, both bonds are locked and the god oracle makes the final call.

Why Resolvers Tell the Truth

  • Wrong resolution → lose bond to the disputer
  • Frivolous dispute → lose bond to the resolver
  • Honest behavior is the only rational strategy
The answer is skin in the game. Anyone can resolve by posting a bond. If disputed, both sides have money on the line — loser's bond goes to the winner. Resolvers are honest because a wrong resolution costs them their bond. Disputers are honest because a frivolous challenge costs them their bond. No protocol fee needed — the economics are self-funding.

pallet-revive Gotchas

Map the account first

32-byte keys don't map to H160 automatically

if (!(await api.isAddressMapped(origin))) {
  await watchTx(mapAccount(typedApi, signer));
}

value is plancks, not wei

divide by NativeToEthRatio before calling the SDK

const nativeValue =
  valueWei / (await typedApi
    .constants.Revive.NativeToEthRatio());
Two gotchas when calling pallet-revive contracts from a Substrate frontend. First: every Substrate account must call map_account once — 32-byte keys don't map to 20-byte H160 addresses automatically. Second: the SDK value field is in native plancks, not wei. Passing parseEther directly asks pallet-revive to move millions of PAS. Keep everything in wei internally and divide by NativeToEthRatio right before calling the SDK.
